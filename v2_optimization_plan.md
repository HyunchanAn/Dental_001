# [기획서] CVM AI 판독 시스템 고도화 및 실무 최적화 로드맵 (V2)

## 1. 개요
현재 구축된 YOLOv8 + EfficientNet 기반의 2단계 파이프라인(Plan 2)은 Kappa 0.63이라는 유의미한 수치를 달성했으나, 실무 적용을 위해 **데이터 편향성 제거, 인접 단계 판별력 향상, 에지 케이스 대응력**을 강화하는 고도화 단계를 추진함.

---

## 2. 핵심 고도화 과제 (3대 중점 사항)

### ① 데이터 샘플링 및 클래스 불균형 해소 (Data Engineering)
*   **문제점:** 특정 성장 단계(예: Stage 3, 4)에 데이터가 쏠려 있어 소수 클래스(Stage 1, 6)에서 오분류 발생 가능성 높음.
*   **해결 방안:**
    *   **Weighted Random Sampling:** 학습 시 부족한 클래스의 데이터를 더 자주 노출시켜 모델의 편향성을 강제로 교정.
    *   **데이터 증강(Augmentation) 차별화:** 소수 클래스에 대해서만 더 강력한 Geometric Augmentation(회전, 전단 변형 등) 적용.
    *   **모니터링 강화:** `check_sampling.py`를 학습 파이프라인에 통합하여 클래스별 성능 변화를 실시간 추적.

### ② 모델 알고리즘 고도화 (Algorithm Refinement)
*   **문제점:** 인접 단계 간의 특징이 모호한 CVM 특성상 단순 분류로는 미세 판독이 어려움.
*   **해결 방안:**
    *   **Ordinal Smoothing 적용:** Label Smoothing을 서열 관계에 맞게 수정하여 인접 단계 오차에 대한 패널티 최적화.
    *   **고해상도 학습 (RTX 5080 활용):** 입력 해상도를 **640px~800px**로 상향하여 경추 하연의 미세 함몰도를 더 정밀하게 포착.
    *   **Soft Labeling 도입:** 전문가의 확신도를 반영한 Soft Target 학습 고려.

### ③ 시스템 견고성 및 사용자 루프 구축 (Robustness & UX)
*   **문제점:** 저품질 영상이나 특이 케이스에서 YOLO 검출 실패 시 대응 로직 부족.
*   **해결 방안:**
    *   **Confidence Scoring:** 모델의 확신도가 낮은 경우 "전문가 확인 필요" 알람 및 사용자 보정 유도.
    *   **Active Learning 루프:** `labeling_tool.py`를 판독 서비스와 연동하여 오답 데이터를 즉시 학습셋으로 피드백하는 구조 구축.

## 3. 실행 로드맵 (Short-term)

| 단계 | 과제명 | 세부 내용 | 목표 지표 |
| :--- | :--- | :--- | :--- |
| **1단계** | **데이터 밸런싱** | Weighted Sampler 도입 및 클래스별 Recall 교정 | 클래스별 Kappa 편차 10% 이내 |
| **2단계** | **고해상도 튜닝** | 입력을 768px로 상향 및 RTX 5080 전용 하이퍼파라미터 적용 | MAE (평균 오차) 0.5 미만 |
| **3단계** | **에지 케이스 대응** | 신뢰도 기반 필터링 및 오답 시각화(`Inference.py` 고도화) | 실무 적용 가능성 검증 |

---

## 4. 기대 효과
*   **임상적 신뢰도 확보:** 인접 단계 간 오차를 최소화하여 실제 교정 진단 보조 도구로 활용 가능.
*   **성능 극대화:** 보유 자산인 RTX 5080과 M2 Pro의 가속 성능을 최대한 활용한 고해상도 판독 실현.
*   **지속 가능한 모델:** 사용자가 사용함에 따라 점진적으로 성능이 개선되는 데이터 선순환 구조 완성.
